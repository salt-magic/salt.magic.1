## 2. SPACING-KONSISTENZ

### Inkonsistenzen gefunden

#### S1 — Section Vertical Padding (HIGH)

Die meisten Sections verwenden `py-[clamp(64px,8vw,100px)]`, aber es gibt Abweichungen:

| Section | Padding | Abweichung |
|---------|---------|------------|
| Standard (meiste) | `py-[clamp(64px,8vw,100px)]` | Basis |
| CtaBanner | `py-[clamp(80px,10vw,120px)]` | +20px min/max |
| SocialProof | `py-[clamp(64px,10vw,100px)]` | Anderer vw-Wert |
| Einige Sections | `pb-[clamp(80px,12vw,140px)]` | Deutlich groesser |

**Fix-Empfehlung:** Einheitlich `py-[clamp(64px,8vw,100px)]` fuer alle Sections. CtaBanner darf als Ausnahme etwas groesser sein, aber andere sollten konsistent bleiben.

#### S2 — Horizontal Padding (MEDIUM)

Drei verschiedene Horizontal-Padding-Werte im Einsatz:

| Variante | Wert | Wo |
|----------|------|-----|
| Standard | `px-[clamp(24px,5vw,64px)]` | Meiste Sections |
| Breiter | `px-[clamp(24px,5vw,80px)]` | Einige Sections |
| Extra breit | `px-[clamp(32px,6vw,80px)]` | Story, Partner |

**Fix-Empfehlung:** Auf maximal 2 Varianten reduzieren — Standard (64px max) und Wide (80px max).

#### S3 — Heading Margin-Bottom (MEDIUM)

Inkonsistente Abstands nach Headlines:

| Wert | Wo |
|------|-----|
| `mb-3` | Einige Sub-Sections |
| `mb-5` | Einige TextBlocks |
| `mb-6` | Verbreitet |
| `mb-8` | Einige grosse Headlines |

**Fix-Empfehlung:** Standardisieren: `mb-5` nach Eyebrow, `mb-6` nach H2-Headlines, `mb-3` nach H3-Sub-Headlines.

#### S4 — Spacer-Divider zwischen Sections (LOW)

In `page.tsx` werden Spacer mit `h-[clamp(32px,4vw,48px)]` verwendet, aber nicht konsequent zwischen allen Sections.

#### S5 — Gap-Werte in Grids (LOW)

Verschiedene Gap-Werte ohne klares System:
- `gap-[clamp(40px,6vw,80px)]` — grosse Layouts
- `gap-[clamp(40px,5vw,64px)]` — mittel
- `gap-[clamp(24px,3vw,40px)]` — kompakt
- `gap-8`, `gap-6`, `gap-4` — feste Werte

## 3. SCHRIFTGROESSEN

### F1 — Zu kleine Schrift auf Desktop (HIGH)

| Element | Groesse | Problem |
|---------|---------|---------|
| Footer Section-Headers | `text-[10px]` | Unter 12px Minimum — schwer lesbar |
| AnnouncementBar Text | `text-[11px]` | Grenzwertig, besonders auf Mobile |
| Blog-Datum Labels | `text-[11px]` | Zu klein fuer wichtige Info |
| Einige Eyebrow-Labels | `text-[11px]` | Grenzwertig |

**Fix-Empfehlung:** Minimum-Schriftgroesse auf 12px anheben. Footer-Headers auf `text-xs` (12px). AnnouncementBar und Blog-Dates auf mindestens 12px.

### F2 — Body-Text Inkonsistenz (MEDIUM)

Verschiedene Body-Text-Groessen ohne klares System:

| Groesse | Wo |
|---------|----|
| `text-[17px]` | Vereinzelt |
| `text-[16px]` | Einige Body-Texte |
| `text-[15px]` | Haeufigster Body-Text |
| `text-[14px]` | Newsletter, Footer |
| `text-[13px]` | Footer Links, CTAs |
| `text-[12px]` | Comparison-Daten |

**Fix-Empfehlung:** Body-Text auf 2-3 Stufen reduzieren:
- Primary Body: `text-[16px]` (Standard-Absaetze)
- Secondary Body: `text-[14px]` (kompakte Bereiche)
- Caption/Meta: `text-[12px]` (Datum, Labels)

### F3 — Mobile Font-Size Skalierung (MEDIUM)

Headlines nutzen `clamp()` — gut. Aber Body-Text bleibt statisch (z.B. `text-[15px]`), was auf kleinen Viewports (320px) zu eng werden kann.

**Fix-Empfehlung:** Body-Text mit `text-[clamp(14px,3.5vw,16px)]` fluid machen.

## 4. ACCESSIBILITY

### A1 — Comparison-Tabelle nicht semantisch (HIGH)

Die Vergleichstabelle (`Comparison.tsx`) nutzt `<div>` Grid statt `<table>`. Screen-Reader koennen Zeilen und Spalten nicht zuordnen.

**Fix:** Umbauen auf `<table>`, `<thead>`, `<tbody>`, `<th scope="col">`, `<td>`.

### A2 — Carousel Slide-Status nicht kommuniziert (MEDIUM)

Hero-Carousel und Testimonials-Carousel zeigen aktiven Slide visuell (Gold-Bar), aber `aria-current="true"` fehlt auf dem aktiven Indikator.

**Fix:** `aria-current="true"` auf den aktiven Slide-Button setzen.

### A3 — Reduced-Motion nicht ueberall respektiert (MEDIUM)

| Component | prefers-reduced-motion | Status |
|-----------|----------------------|--------|
| Hero | Ja (useReducedMotion) | OK |
| WhySection | Ja | OK |
| StorySection | Ja | OK |
| SocialProof | Ja | OK |
| AnnouncementBar Marquee | Ja (CSS) | OK |
| Testimonials Carousel | **Nein** | FEHLT |
| StickyMobileCta | **Nein** | FEHLT |

**Fix:** `useReducedMotion()` Hook in Testimonials und StickyMobileCta einbauen. Carousel-Auto-Rotation bei reduced-motion deaktivieren.

### A4 — Skip-Link vorhanden (OK)

`layout.tsx` hat einen sr-only Skip-to-Content Link — korrekt implementiert.

### A5 — Touch-Targets (OK)

Alle Buttons haben `min-h-[44px] min-w-[44px]` — WCAG-konform.

### A6 — Focus-Visible (OK)

Konsistent auf allen interaktiven Elementen: `focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-4 focus-visible:outline-gold`.

## 5. RESPONSIVE DESIGN

### R1 — Hidden Elements (LOW)

Einige Desktop-Features werden auf Mobile komplett versteckt statt adaptiert:
- Arrow-Buttons (Hero, Testimonials): `hidden md:flex`
- Film-Grain-Texture: `hidden md:block`

Dies ist akzeptabel fuer Performance, aber alternative Touch-Gesten (Swipe) sollten dokumentiert/implementiert sein.

### R2 — Mobile Menu (OK)

Fullscreen-Overlay mit zentrierten Links, korrektes `overflow: hidden` auf Body, korrekte z-Index-Hierarchie.

### R3 — Breakpoint-Strategie (OK)

Mobile-first mit `md:` (768px) und `lg:` (1024px) Breakpoints. Fluid clamp()-Werte verhindern Layout-Spruenge.

## 6. WEITERE UX-ISSUES

### U1 — `transition-all` in link-underline (LOW)

In `globals.css` nutzt `.link-underline` noch `transition-all` statt spezifischem `transition-[text-decoration-color]`. Performance-Impact minimal, aber inkonsistent mit dem Rest der Codebase wo `transition-all` bewusst entfernt wurde.

### U2 — Hardcoded Colors (MEDIUM)

Mehrere Components nutzen hardcoded rgba()-Werte statt Tailwind-Tokens:
- Hero: `rgba(0,0,0,0.10)` bis `rgba(0,0,0,0.88)` (Gradient)
- CtaBanner: `rgba(26,50,72,0.88)` (Gradient)
- StorySection: `bg-[#1D3347]` statt `bg-footer-dark`

**Fix-Empfehlung:** `#1D3347` ist identisch mit `footer-dark` — Token verwenden. Gradients koennen hardcoded bleiben, aber in einem Kommentar dokumentieren.

### U3 — CTA-Button-Groessen inkonsistent (LOW)

| Kontext | Padding | Text-Groesse |
|---------|---------|--------------|
| Hero Primary | `px-9 py-4` | `text-[11px]` |
| Section CTAs | `px-6 py-2.5` | `text-[13px]` |
| Mobile Sticky | `py-3.5` | `text-[13px]` |

**Fix-Empfehlung:** CTA-Buttons in 2 Groessen standardisieren: Large (Hero, Section-End) und Compact (Inline, Mobile).

### U4 — Newsletter Formular nur Frontend (INFO)

Newsletter-Section hat kein Backend — Submit-Handler ist leer. Sollte mindestens eine visuelle Bestaetigung oder Hinweis zeigen.

### U5 — Mobile Safe-Area Bottom Spacer (OK)

`StickyMobileCta` nutzt `env(safe-area-inset-bottom)` — korrekt fuer Notch-Devices.

### U6 — Kein Lazy-Loading auf Below-Fold Images (LOW)

Einige Bilder unterhalb des Folds (Story, ForEveryone) koennten `loading="lazy"` oder `priority={false}` (Next.js) nutzen fuer bessere Performance. Next.js Image ist standardmaessig lazy — pruefen ob `priority` falsch gesetzt ist.
