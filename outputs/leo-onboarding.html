01

Before we start — 2 minute check

You already have the Claude Desktop App installed. Before our session, please confirm one thing:

Quick check

Open the Claude Desktop App. In the top-left corner of the sidebar, do you see a tab labeled "Code" with a </> icon?

Yes? You're good. Close the app, we'll reopen it together.
No? Message Kerstin before our session — we need a different setup path.

What does each tool do? (quick overview)

Tool What it does
Claude Desktop App Your AI assistant. You describe changes, it makes them.
Git Tracks every version of the code, like a time machine.
Node.js Runs the website locally on your laptop so you can preview changes.
GitHub The cloud where the website's code lives. Everyone syncs through it.
Vercel Automatically publishes your changes to salt-magic.com every time you push.

The good news: Claude will install and configure most of these for you. You don't need to download anything manually.

02

Open the Code tab in the Desktop App

1

Launch the Claude Desktop App

Find it in your applications / Start menu and open it.

2

Click the Code tab

In the top-left sidebar, click the </> Code tab. You'll see a dashboard with your stats.

3

Find the "Select folder..." button

Look at the bottom of the screen, right above the chat input field. You'll see two small buttons: Local and Select folder...

Good to know

The bottom chat input is where you'll spend most of your time. Everything you want Claude to do, you type here in plain language — English or German, doesn't matter.

03

Create your project folder

Before we connect anything, you need an empty folder on your Desktop where the website's code will live.

1

Go to your Desktop

Minimize all windows so you can see your Desktop background.

2

Create a new folder

Right-click on empty space → New → Folder. Name it salt.magic.1. Press Enter.

3

Back to the Desktop App

Click Select folder... at the bottom, navigate to your Desktop, pick the salt.magic.1 folder you just created, and confirm.

What you'll see

The button at the bottom now shows salt.magic.1 instead of "Select folder...". This means Claude knows where to work. The folder name is also your visual confirmation that you're in the right project.

04

Let Claude install the development tools

Now for the magic part. Claude will install Git, Node.js, and GitHub CLI for you. You don't need to download anything manually.

Type this in the chat

Please check if git, node.js, and github cli are installed on this machine. If anything is missing, install it. Report back what you did.

Press Enter. Wait.

What you'll see

  • Claude will run a check (takes 10 seconds)
  • If anything is missing, it will start installing it — this can take 2–5 minutes per tool
  • Windows may show a popup asking "Do you want to allow this app to make changes?" — click Yes
  • When done, Claude reports the versions it found/installed
Be patient

If Claude says "installing Node.js, this will take 2–3 minutes" — trust it. Don't click around, don't cancel. Go get a coffee. The screen may look frozen, but it's working.

05

Configure your Git identity

Every change you make gets a name tag — so others can see "Leo made this change on Thursday at 3pm". You set this up once, then forget about it.

Use your GitHub email

The email you use here must match an email verified on your GitHub account. Otherwise your commits appear in the repository but won't be linked to your GitHub profile (no avatar, no contributor stats). If you're not sure which email you used, check at github.com/settings/emails.

Type this in the chat

Please set my git identity globally: - Name: Leo [your surname] - Email: saltmagic.th@gmail.com Then confirm by showing me the output of "git config --global --list | grep user".

Replace [your surname] with your actual surname. The email saltmagic.th@gmail.com is the one registered with your GitHub account — leave that as is. Press Enter.

Don't skip this

If you skip this step, your first commit will fail with an "Author identity unknown" error. It's a one-line fix but confusing for beginners.

06

Log into GitHub

GitHub is the cloud storage for the website's code. You need to prove it's really you before uploading changes.

Type this in the chat

Please log me into GitHub via the gh CLI.

What happens next

  1. Claude will show you an 8-digit code (looks like A1B2-C3D4) and a link to github.com/login/device
  2. Click the link — your browser opens
  3. Log into GitHub with your own account (not someone else's)
  4. Paste the 8-digit code into the browser
  5. Click "Authorize"
  6. Come back to the Desktop App — Claude will say "Authentication complete"
Do this once, remember forever

Your laptop now remembers your GitHub login. You'll never have to do this again — Claude can push changes on your behalf indefinitely.

07

Clone the Salt.Magic repository

"Clone" means: download a copy of the website code from GitHub to your laptop. After this step, you have the entire project locally.

Type this in the chat

Please clone the repository https://github.com/salt-magic/salt.magic.1.git into this folder. The repo contents should end up directly in the current folder. After cloning, show me the output of "git log -3 --oneline".

What to expect

  • Claude may use a clone technique called "init + pull" because the folder already has a small hidden .claude/ folder. This is normal and expected.
  • After cloning, you'll see the last 3 commits — dates, authors, and short messages
  • The top entry is the most recent change anyone made
You may notice

The title bar at the top of the Desktop App might change from salt.magic.1 to show the repo name. Your folder on disk still has the same name — it's just a cosmetic detail.

08

Start the local development server

This runs the Salt.Magic website on your own laptop, in your browser. You'll use this to preview changes before pushing them live.

Type this in the chat

Please do the following: 1. Change into the "site" folder 2. Run "npm install" to install dependencies (this may take 2-3 minutes the first time) 3. Start the dev server with "npm run dev" 4. Tell me the URL to open in my browser

What to expect

  • Phase 1 (npm install): 2–3 minutes. Claude will download hundreds of small packages.
  • Phase 2 (dev server): 30 seconds. Claude starts Next.js.
  • You'll get a URL — usually http://localhost:3000

Open the URL in your browser

Copy the URL Claude gives you, paste it into a new browser tab. You should see the Salt.Magic website running locally.

Port already in use?

If localhost:3000 is taken (e.g., another project running), Next.js will automatically use 3001, 3002, etc. Always use the URL Claude tells you in the chat — don't hardcode :3000.

When you're done for the day

Tell Claude: "Stop the dev server". Otherwise it keeps running in the background.

09

Your first edit — with safety net

You've set everything up. Now make a real change and see it live (locally) without touching the production website. Think of this as a dress rehearsal.

Step 1 — Ask for a visible change

Please make a small visible test change: in the Hero component, append "(TEST)" to the main headline. Work directly on main, no side branch. Do not commit or push yet - I just want to see it in my browser first.

Claude edits the file. Within 1–2 seconds, your browser automatically refreshes — you see "(TEST)" appended to the Hero headline. This is called Hot Reload.

What just happened

You didn't refresh the browser. Next.js detected the file change and updated the page for you. This is why local development is so fast — you see changes instantly.

Step 2 — Ask Claude to undo

Revert that change completely. Remove the "(TEST)" from the Hero. After reverting, run "git status" and confirm there are no uncommitted changes.

Browser updates. "(TEST)" is gone. Claude confirms working tree clean.

Your golden rule

As long as you haven't pushed yet, every change is reversible. If something doesn't look right: just say "Undo the last change" or "Revert to what we had before". Claude handles it.

10

Commit, push, watch it go live

Time to make a real change — one that actually goes live. We'll keep it tiny and easy to reverse if needed.

Point of no return

Once you push, the change is public on GitHub and salt-magic.com will rebuild within 60 seconds. Reversing it requires a second commit. Keep your first real change small.

Step 1 — Make a small, visible change

Pick something harmless. Example: append a tiny marker to the page footer or a blog post you wrote. Then:

Please add the text "[first edit test by Leo]" to the very end of CLAUDE.md. Work directly on main, no side branch. Show me the diff so I can verify before we commit.

CLAUDE.md doesn't affect the live website — safe first commit.

Step 2 — Commit and push

Looks good. Please do the following: 1. Stage the change 2. Commit with the message: "First edit by Leo" 3. Push to origin main 4. Show me "git log -3 --oneline" so I can see my commit at the top

Step 3 — Verify on the live site

  1. Wait ~60 seconds (Vercel needs to build)
  2. Open salt-magic.com
  3. The site loads normally — this is the success signal
  4. Your commit didn't visually change anything (we only edited CLAUDE.md), but it did trigger a successful deployment. You just pushed to production.
Celebrate

You just made and deployed your first change to the Salt.Magic website. From here on, every edit follows the same pattern: describe the change to Claude → preview in browser → commit → push → Vercel does the rest.

11

Changing text on the website

Want to tweak a headline? Fix a typo? Rewrite a whole section? You don't touch any files yourself. You describe what you want in plain English, and Claude makes the edit. Here's the pattern.

Step 1 — See it before you change it

Open the live site (salt-magic.com) or your local preview (localhost:3000). Find the text you want to change. Note which page it's on (Home, Partner, Blog, Privacy) and which section (Hero, Formula, FAQ, Footer, etc.).

Step 2 — Type this in the chat

On the homepage, in the [Hero / Formula / FAQ / ...] section, change this text: "[paste the exact current text here]" to: "[paste the new text here]" Work directly on main, no side branch. Show me the change in my browser first - don't commit yet.
Copy the exact current text

The easier you make it for Claude to find the text, the faster it edits. Copy-paste the current sentence from your browser. Don't paraphrase.

Step 3 — Verify in the browser

Your local preview auto-refreshes within 1-2 seconds. Read the new text in context. If it doesn't feel right, just say:

Make it shorter / punchier / more formal / friendlier.

Iterate as many times as you want — nothing is live yet.

Step 4 — Commit and push

Once happy:

Looks good. Please commit with the message "Update [section] copy" and push to origin main.

Live in ~60 seconds on salt-magic.com.

What you can change this way

Any visible text: headlines, paragraphs, button labels, FAQ answers, product descriptions, footer links, meta titles (the tab name in the browser), SEO descriptions (what Google shows in search results).

What you should NOT change casually
  • Prices (e.g., "290 THB") — these are tied to Lazada listings
  • Legal pages (Privacy Policy, Terms) — review by a lawyer first
  • Product claims ("FDA approved", "pharmaceutical-grade") — FDA approval is still in progress, be careful with claims
12

Changing photos on the website

Replacing an image is two moves: (1) get your new photo into the project folder, (2) tell Claude to swap it. Claude handles the rest — resizing, converting to WebP (a modern fast format), SEO-friendly naming, and updating the code. You never open Photoshop.

Why WebP?

WebP is 30-60% smaller than JPEG or PNG at the same quality. Faster-loading website = better Google ranking + happier visitors. Our optimizer script does this automatically.

Step 1 — Drop your photo into the "new" folder

In your file explorer (Windows Explorer or Mac Finder), navigate to:

salt.magic.1 / site / public / images / new

If the new folder doesn't exist yet, create it. Copy or drag your photo into that folder. It can be any format (JPG, PNG, HEIC from your iPhone, etc.) and any size — even a 10 MB original from your camera is fine.

Descriptive filenames help

Instead of IMG_2847.jpg, rename to leo-holding-sachet-beach.jpg before dropping it in. This becomes part of the SEO-friendly final filename.

Step 2 — Tell Claude which image to replace

I've dropped a new photo in site/public/images/new/ called "[your-filename.jpg]". Please replace the [Hero sachet image / WhySection photo / Glass Jar product photo / ...] with this new one. Alt-text for SEO: "[one sentence describing what's in the photo, using 1-2 natural keywords - see examples below]" After the swap: 1. Optimize the new image (WebP, max 2000px wide, quality 88) 2. Move it to the correct production folder 3. Update the component to use the new file 4. Delete the original from public/images/new/ 5. Show me the page in my browser so I can verify

Step 3 — What Claude does behind the scenes

  1. Runs the Salt.Magic image optimizer (scripts/optimize-images.py) — resizes, strips watermarks if any, converts to WebP
  2. Moves the optimized file into the right folder (e.g., site/public/images/products/)
  3. Updates the React component (e.g., Hero.tsx) with the new path and your alt-text
  4. Hot-reloads your browser so you see the change immediately
First-time setup

The very first time you swap a photo, Claude may need to install three Python libraries (Pillow, opencv-python, numpy). This takes 1-2 minutes. Only happens once — every swap after that is instant.

Step 4 — Writing good alt-text (SEO)

Alt-text is the sentence screen readers read aloud for blind users, and what Google uses to understand your image. Two rules:

  • Describe what's in the photo (first priority)
  • Weave in 1-2 natural keywords (without stuffing)

Good examples:

  • "Salt.Magic travel sachet next to coconut water on a Koh Samui beach"
  • "Glass jar of natural electrolyte powder on a linen cloth"
  • "Woman in yoga pose drinking mineralized water, Koh Samui wellness retreat"

Bad examples:

  • "electrolytes thailand best natural hydration buy online" — keyword stuffing, Google penalizes this
  • "image.jpg" — tells Google nothing

Step 5 — Commit and push

Once you've confirmed the new photo looks right in your browser:

Perfect. Commit with the message "Replace [which image]" and push to origin main.

Live on salt-magic.com in ~60 seconds.

Don't commit the "new" folder

The site/public/images/new/ folder is a staging area only. Claude should always delete the original file after optimization. If you ever see it still sitting in new/ after a swap, tell Claude: "Delete the original from public/images/new/".

Swap several photos at once

Drop multiple images into new/, then tell Claude all the swaps in one message. Example: "Replace the Hero with new/beach-sunset.jpg, the WhySection photo with new/yoga-stretch.jpg, and the Glass Jar product shot with new/jar-white-bg.png." Claude batches the work.

If something goes wrong

Claude asks for permission before running something

This is a feature, not a bug. The Desktop App asks before destructive actions (deleting files, moving folders, force-pushing). Read what Claude wants to do, then:

  • Allow once — if you want to permit just this one command (usually the right choice)
  • Always allow — only if you're sure you always want to allow this type of action
  • Deny — if something seems off; Claude will stop and explain

Your browser shows the old version even after a change

Press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac) for a hard refresh.

Claude stopped responding

Click + New session in the left sidebar, re-select your salt.magic.1 folder, and continue.

Slash commands (/create-plan etc.) don't work

The Desktop App doesn't always recognize them. Workaround: describe what you want in plain language, e.g., "Follow the instructions in .claude/commands/create-plan.md for this change".

"Something is weird, I don't know what happened"

Please run "git status" and "git log -5 --oneline". Summarize what state the project is in and if there's anything unexpected.

Claude will diagnose and tell you how to recover.

Side branch appeared, you just want to work on main

If you see main ← claude/xyz-abc at the bottom of the chat and you didn't want a branch, say:

Please clean up the experimental branch and put me back on main with a clean working tree. Show me "git status" afterwards.

Empty folder stuck on Windows

If Claude says "I couldn't fully delete a folder because Windows has a lock on it" — it's harmless. Once the Desktop App closes, you can delete the empty folder manually via File Explorer.

The new photo didn't replace the old one

Your browser might be caching the old image. Hard-refresh with Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac). If still wrong, ask Claude:

Show me which image file the [Hero / WhySection / ...] component is actually importing right now. If it's still the old one, fix the import.

My new photo looks blurry or pixelated

Usually means the original was too small. The optimizer resizes down but never up. Minimum source: 2000px wide for hero images, 1200px wide for card images. If your source is smaller, ask Claude:

The new photo looks pixelated. How wide is the source file I dropped in? If it's under 2000px, tell me and I'll provide a higher-resolution version.

I changed some text but it still shows the old version

Two causes:

  • Browser cache — hard-refresh with Ctrl+Shift+R / Cmd+Shift+R
  • Not yet deployed — pushes take ~60 seconds to go live. Check vercel.com or just wait a minute

If neither works:

Please confirm: was my last text change actually committed and pushed? Show me "git log -3 --oneline" and the relevant line from the component file.
